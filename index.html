Desarrollo práctico — Rutas estáticas (Packet Tracer)

Caso del Capítulo 2. Resuélvelo en papel / Packet Tracer y luego abre la solución para comparar.

Tres routers (R1 – R2 – R3) conectados por enlaces seriales WAN, cada uno con una LAN. Configure el direccionamiento y las rutas estáticas necesarias para que haya comunicación de extremo a extremo entre todas las PC.

  1. Configure nombre de host, contraseña enable secret y banner en cada router.
  2. Configure y active todas las interfaces (recuerde clock rate en los extremos DCE).
  3. Configure las rutas estáticas necesarias: una ruta por defecto en R1 y una ruta resumida /22 en R3.
  4. Verifique con show ip route y demuestre con ping entre PC1↔PC3 y PC2↔PC3.

1) Configuración de R1 (router stub → ruta por defecto)

R1(config)# hostname R1 R1(config)# enable secret class R1(config)# interface fa0/0 R1(config-if)# ip address 172.16.3.1 255.255.255.0 R1(config-if)# no shutdown R1(config-if)# interface s0/0/0 R1(config-if)# ip address 172.16.2.1 255.255.255.0 R1(config-if)# clock rate 64000 ! R1 es DCE R1(config-if)# no shutdown R1(config)# ip route 0.0.0.0 0.0.0.0 172.16.2.2 ! ruta por defecto

2) Configuración de R2 (router intermedio)

R2(config)# interface fa0/0 R2(config-if)# ip address 172.16.1.1 255.255.255.0 R2(config-if)# no shutdown R2(config-if)# interface s0/0/0 R2(config-if)# ip address 172.16.2.2 255.255.255.0 R2(config-if)# no shutdown ! DTE: sin clock rate R2(config-if)# interface s0/0/1 R2(config-if)# ip address 192.168.1.2 255.255.255.0 R2(config-if)# clock rate 64000 ! R2 es DCE hacia R3 R2(config-if)# no shutdown R2(config)# ip route 172.16.3.0 255.255.255.0 serial0/0/0 R2(config)# ip route 192.168.2.0 255.255.255.0 192.168.1.1

3) Configuración de R3 (ruta resumida /22)

R3(config)# interface fa0/0 R3(config-if)# ip address 192.168.2.1 255.255.255.0 R3(config-if)# no shutdown R3(config-if)# interface s0/0/1 R3(config-if)# ip address 192.168.1.1 255.255.255.0 R3(config-if)# no shutdown ! DTE: sin clock rate R3(config)# ip route 172.16.0.0 255.255.252.0 192.168.1.2 ! /22 cubre 172.16.0–3.0

4) Verificación

  • show ip interface brief → todas las interfaces deben estar up / up.
  • show ip route → en cada router deben aparecer las 5 redes (C de conectadas, S de estáticas).
  • ping de PC1 a 192.168.2.10 y de PC2 a 192.168.2.10 → !!!!! = éxito.
Por qué estas rutas: cada router solo conoce sus redes conectadas. R1 es stub (solo va a R2) → ruta por defecto. R3 usa una ruta resumida /22 que agrupa 172.16.0.0–172.16.3.0 en una sola línea. R2 está en el medio, por eso necesita rutas hacia ambos extremos. Si un ping falla, casi siempre es por una ruta de regreso faltante o una interfaz en down.
